## 3. Заставка и авторизация

### 3.1 Splash Screen

**Визуальное описание:**

Полноэкранный однотонный фон (цвет `scaffoldBackgroundColor` из активной темы). Логотип и текст появляются из слегка опущенной позиции с одновременным fade-in.

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│                                 │
│          ┌───────────┐          │
│          │  [LOGO]   │          │  SvgPicture.asset, 120×120px
│          │  SVG file │          │  (tezyubor_logo.svg)
│          └───────────┘          │
│                                 │
│        tez  YUBOR               │  RichText, fontSize: 42, fontWeight w800
│        ───  ──────              │  letterSpacing: -1.5
│   "tez" = тема текст (белый/тёмн│  "tez" = Colors.white (тёмная тема)
│   "YUBOR" = #FF6929 оранжевый  │          или Color(0xFF1a1a18) (светлая)
│                                 │  "YUBOR" = AppColors.primary (#FF6929)
│                                 │
└─────────────────────────────────┘
```

**Анимация:**
- Два контроллера: `FadeTransition` (opacity 0→1) + `SlideTransition` (от `Offset(0, 0.08)` до `Offset.zero`)
- Длительность: **700ms**, кривая `easeOut`
- Оба запускаются одновременно при initState
- После анимации — авто-переход (без задержки, перенаправление зависит от токена)

**Логика авто-редиректа:**
- Нет токена → Login Screen (режим Аптека по умолчанию)
- Токен есть + роль Admin → Admin ShellRoute (с нижней навигацией)
- Токен есть + роль Pharmacy + нет сохранённой локации → Location Setup Screen
- Токен есть + роль Pharmacy + есть локация → Pharmacy ShellRoute (с нижней навигацией)

│        tez  YUBOR               │
│                                 │
│      Вход в систему             │  Title Large, жирный — меняется по режиму
│      Быстрая доставка лекарств  │  Body Medium, второстепенный цвет

│  ┌─────────────────────────┐    │
│  │ 👤  Логин               │    │  Outlined TextField, prefix icon
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 🔒  Пароль           👁 │    │  Пароль с кнопкой видимости суффикс
│  └─────────────────────────┘    │

│  ┌─────────────────────────┐    │  Показывается только при ошибке

**Два режима (переключается 5 тапами по логотипу):**

| Элемент | Режим Аптека | Режим Администратор |

| Кнопка | "Войти" | "Войти как администратор" |

| Состояние | Визуал |
|---|---|

### 4.1 Экран Заказы

#### Общая структура

```
┌─────────────────────────────────┐
│  Заказы                🔍  🔽  │  AppBar
│                                 │  🔍 = search icon, 🔽 = filter_list (с Badge)
├─────────────────────────────────┤
│ [Все][Ожидает][Ожид.подтв.][..] │  TabBar (горизонтальный скролл, 9 вкладок)
├─────────────────────────────────┤
│  [Курьер: Яндекс ×] [Apr ×]    │  _ActiveFilterRow (только при активных фильтрах)
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐    │
│  │   Карточка заказа       │    │
│  └─────────────────────────┘    │
│  ┌─────────────────────────┐    │
│  │   Карточка заказа       │    │
│  └─────────────────────────┘    │
│  ...                            │
│                                 │
│                  ┌──────────┐   │
│                  │+ Заказ   │   │  FAB (оранжевый, extended)
│                  └──────────┘   │
├─────────────────────────────────┤
│  📋  📊  👥  ⚙️                 │
└─────────────────────────────────┘
```

#### AppBar — кнопки

**Кнопка Поиск (`Icons.search`):**
- При нажатии AppBar полностью заменяется TextField в то же пространство
- Поле появляется анимированно (fade)
- Placeholder: "Поиск заказов..."
- Кнопка ← закрывает поиск и очищает query
- Поиск по токену заказа и имени клиента
- **Debounce: 400ms** — запрос отправляется через 400ms после последнего нажатия

**Кнопка Фильтр (`Icons.filter_list`):**
- Обёрнута в `Badge`: оранжевый кружок показывается когда активен хоть один фильтр по курьеру ИЛИ дате
- При нажатии открывает Filter Bottom Sheet

#### _ActiveFilterRow — строка активных фильтров

Появляется между TabBar и списком только если есть активные фильтры. Горизонтальный скролл чипов.

```
┌─────────────────────────────────┐
│  [Яндекс ×]  [1 апр – 30 апр ×]│  Чипы с кнопкой удаления
└─────────────────────────────────┘
```

- Каждый выбранный курьер — отдельный чип с × (удаляет только этот курьер)
- Если задан диапазон дат — один чип с диапазоном + × (очищает обе даты)
- padding: left 16, vertical 4

#### TabBar — 9 вкладок статусов

```
[Все] [Ожидает] [Ожид. подтв.] [Подтверждён] [Курьер едет] [Курьер забрал] [Доставка] [Доставлен] [Отменён]
```

- `TabBar` с `isScrollable: true`, вкладки `Tab(text: ...)`

## 7. Системные состояния

### Состояния экранов со списками

| Состояние | Визуал | Условие |
|---|---|---|
| Loading (первый раз) | `CenteredLoader` — оранжевый спиннер по центру | `isLoading && items.isEmpty` |
| Loading (refresh) | Pull-to-refresh индикатор сверху | `isLoading && items.isNotEmpty` |
| Error (первый раз) | `AppErrorWidget` — иконка + текст + кнопка Повторить | `error != null && items.isEmpty` |
| Empty (нет данных) | `EmptyState` — иконка + заголовок + подсказка | `!isLoading && items.isEmpty` |
| Empty с фильтром | `EmptyState` + кнопка "Сбросить фильтры" | `isFiltered && items.isEmpty` |
| Данные | `ListView.separated`, `RefreshIndicator` | `items.isNotEmpty` |

### Состояния форм

| Состояние | Визуал |
|---|---|
| Idle | Поля пустые/заполненные, кнопка оранжевая активна |
| Validation error | Красный текст под полем, кнопка остаётся активной |
| Submitting | Кнопка = спиннер + disabled, все поля disabled |
| Server error | Красный Container-баннер под формой (с иконкой ✕ для закрытия) |
| Success | Лист закрывается, список обновляется, SnackBar с подтверждением |

### Debounce (задержки поиска)

| Экран | Debounce |
|---|---|
| Заказы (поиск) | 400ms |
| Клиенты (поиск по телефону) | 500ms |
| Бизнесы (поиск аптек) | 400ms |
